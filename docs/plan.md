# Maintenance Report System Plan

## Goal
Build a web app for managing maintenance reports in a simple and professional way.

## MVP users
- Supervisor
- Technician

## MVP features
- Create a maintenance report
- View all reports
- View report details
- Edit a report
- Change report status
- Search and filter reports
- View a basic dashboard

## Main entities
- Reports
- Assets
- Users (basic for now)

## Main pages
- Dashboard
- Reports List
- Create Report
- Report Details
- Edit Report

## Backend scope
- Reports CRUD
- Basic assets support later
- Authentication later

## Database for MVP
- SQLite

## Definition of done
- User can create reports
- User can list reports
- User can view report details
- User can update reports
- User can filter reports
- Data is saved in the database
- Frontend connects to backend successfully
- App can be deployed

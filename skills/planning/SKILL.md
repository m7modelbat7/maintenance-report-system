# Planning Skill

## Goal
Create a small, safe plan before changing code.

## Instructions
1. Restate the task simply.
2. Identify the likely files to change.
3. Propose the smallest implementation path.
4. List acceptance criteria.
5. Do not start coding until the plan is clear.

## Output format
- Task summary
- Files to touch
- Plan
- Risks
- Acceptance criteria
